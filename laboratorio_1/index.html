¿Cuál es la diferencia entre Internet y la World Wide Web?:

El Internet es la red global de computadoras, mientras que la World Wide Web es una de las muchas aplicaciones que se ejecutan en esta red, lo que permite el acceso y la visualización de información a través de páginas web.

¿Cuáles son las partes de un URL?:

Un URL normalmente consta de un esquema, nombre de dominio, ruta y, en algunos casos, puede incluir parámetros y fragmentos.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?:

GET se utiliza para solicitar datos de un recurso específico en el servidor.
HEAD es similar a GET, pero solo devuelve los encabezados de la respuesta, sin el cuerpo del mensaje.
POST se utiliza para enviar datos al servidor para que sean procesados.
PUT se utiliza para enviar datos al servidor para actualizar o crear un recurso en una ubicación específica.
PATCH es similar a PUT, pero se utiliza para realizar modificaciones parciales en un recurso en lugar de reemplazarlo completamente.
DELETE se utiliza para eliminar un recurso específico en el servidor.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?:

Se usa el metodo POST debido a que hace los datos menos visibles al enviar los datos al cuerpo de la solicitud HTTP, permite enviar datos más grandes como contraseñas mas largas y la solicitudes POST no se almacenan en el cache.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?:

El método GET.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?:

No, el código 200 significa que la solicitud se a completado con éxito.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?:

Si, debido a que usualmente los errores 404 se deben a enlaces rotos o paginas eliminadas por el desarrollador, además de que daña el ranking y la reputación del sitio.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?:

Si, debido a que el error 500 inidica un problema interno del servidor que impide que se sirva el contenido solicidado y no se procese la solicitud del usuario, lo que significa que no es responsabilidad del cliente y el desarrollador debe solucionarlos ya que estos también pueden ser resultado de vulnerabilidades de seguridad.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados:

Significa que es obsoleto y que puede ser no compatible con las ultimas versiones de HTML, algunos elementos desaprobados son font, center, strike y acronym.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?:

HTML5 tiene una semántica mejorada, incluye elementos nativos como audio y video, formularios mejorados y mayor compatibilidad con dispositivos moviles.

¿Qué componentes de estructura y estilo tiene una tabla?:

Los componentes principales son tr, th, td, thead, tbody, tfoot, etc. Mientras que en CSS tiene reglas de estilo que cambian propiedades de tipo tabla, celda, fila, encabezado, etc.

¿Cuáles son los principales controles de una forma HTML5?:

Los principales son el elemento input y el control type, el cual puede ser text, password, email, tel, number, etc, lo que permite al usuario ingresar datos de tipos diferentes. Además se encuentran el select, textarea, button para poder crear un formulario.

¿Cuál es el ciclo de vida de los sistemas de información?:

Es un proceso que describe el uso de un sistema desde su concepcion hasta su retiro. Consta de diversas etapas, las cuales son: Identificacion de necesidades y análisis, diseño, desarrollo e implementacion, operacion y mantenimiento, evaluacion y mejora y retiro o desuso.

¿Cuál es el ciclo de desarrollo de sistemas de información?:

Es un ciclo que describe al proceso para crear sistemas eficaces y funcionales. Consta de las siguientes etapas, las cuales son: Planificacion y análisis de requerimientos, diseño del sistema, implementacion y desarrollo, despliegue y puesta en marcha, operacion y mantenimiento y evaluación y retroalimentacion.
